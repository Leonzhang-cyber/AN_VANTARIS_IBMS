# standards

## Purpose

Defines normative cross-contract standards (versioning, identifiers, trace, time, error/status behavior).

## Authority role

This directory is authority guidance for how all contract artifacts must be authored and evolved.

## What belongs here

- versioning standards
- identifier and trace standards
- timestamp standards
- error and status standards

## What must not be placed here

- runtime package implementation code
- environment-specific deployment config
- secret values or private credentials
