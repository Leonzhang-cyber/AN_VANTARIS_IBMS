# canonical

## Purpose

Defines shared VANTARIS ONE domain object semantics across UFMS, EDGE/LINK, Code, DB, Console, and NexusAI.

## Authority role

Canonical definitions here govern field meaning and ownership boundaries for downstream schemas and APIs.

## What belongs here

- canonical object model docs
- module profile mapping docs
- cross-domain ownership semantics

## What must not be placed here

- DB table definitions or migration scripts
- runtime adapters/core implementation details
- package-internal code behavior not part of published contracts
