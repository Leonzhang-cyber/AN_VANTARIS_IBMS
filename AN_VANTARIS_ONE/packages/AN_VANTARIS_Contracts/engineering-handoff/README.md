# engineering-handoff

## Purpose

Provides engineer-facing integration guidance for consuming authority contracts safely.

## Authority role

This directory translates authority contracts into rollout-ready handoff guidance without redefining contracts.

## What belongs here

- engineer onboarding readme
- integration checklists
- handoff sequencing and validation guidance

## What must not be placed here

- unofficial contract forks
- runtime source code snippets treated as authority
- secrets or tenant-specific operational data
