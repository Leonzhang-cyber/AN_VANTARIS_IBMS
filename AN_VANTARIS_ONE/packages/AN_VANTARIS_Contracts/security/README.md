# security

## Purpose

Defines contract-level machine identity, signature, trace, and trust boundary standards.

## Authority role

Security contracts here provide authoritative interoperability rules for EDGE/LINK/Code integrations.

## What belongs here

- machine identity standards
- signature and replay-protection standards
- trust-domain and key-id conventions

## What must not be placed here

- private keys or real secret material
- environment-specific IAM implementation details
- runtime-specific crypto wiring code
