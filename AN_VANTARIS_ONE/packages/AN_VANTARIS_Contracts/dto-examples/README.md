# dto-examples

## Purpose

Holds contract-aligned payload examples for integration and validation handoff.

## Authority role

Examples are derived from authority schemas and OpenAPI docs and must remain version-aligned.

## What belongs here

- example request/response payloads
- edge-to-link handoff examples
- link-to-code delivery and ack examples

## What must not be placed here

- real secrets, credentials, or production identifiers
- examples that diverge from released contracts
- runtime-generated snapshots treated as source of truth
