# ESG A0 Module Rules

1. ESG is a VANTARIS ONE business module.
2. ESG owns sustainability interpretation, not Edge/Link/Contracts.
3. ESG must consume Shared Foundation data through ONE adapter/contracts.
4. ESG may consume UFMS fault intelligence output only through approved boundary.
5. ESG may consume MMS maintenance outcome references only through approved module boundary.
6. ESG must not import UFMS backend/runtime source.
7. ESG must not redefine global Contracts.
8. ESG must not implement protocol drivers.
9. ESG must preserve tenantId / projectId / siteId.
10. ESG must preserve messageId / traceId / correlationId when using events/alarms/fault outputs.
11. ESG runtime implementation requires separate ESG-A1 approval.
12. Any menu/API/DB change requires separate task approval.
