# Module Manifest A0 Rules

1. Every VANTARIS ONE module must have a moduleId.
2. Every module must declare what it owns.
3. Every module must declare what it does not own.
4. Every module must declare consumed objects.
5. Every module must declare provided objects.
6. Every module must declare traceability requirements.
7. No ONE module may privately implement Edge runtime.
8. No ONE module may privately implement Link runtime.
9. No ONE module may fork global Contracts.
10. No ONE module may import UFMS runtime source.
11. Any runtime implementation requires A1 approval.
12. Any API/DB/menu/RBAC change requires explicit task approval.
13. Any future module must be registered in manifest baseline before implementation.
