# ONE Shared Foundation Consumer A0

## 1. Decision

VANTARIS ONE / IBMS will consume UFMS-led Shared Foundation assets instead of independently developing duplicate Edge / Link / Contracts runtimes.

Shared Foundation assets:

- AN_VANTARIS_Contracts
- AN_VANTARIS_EDGE
- AN_VANTARIS_LINK

## 2. Reason

UFMS has independent product and sales potential.
Edge / Link / Contracts are core foundations required by UFMS and VANTARIS ONE.
Developing them first under UFMS-led Shared Foundation accelerates UFMS productization and avoids duplicate implementation.

## 3. Immediate Effect

Pause the following ONE-private tasks:

- EDGE-A2-PROTOCOL-PLUGIN-DESCRIPTOR-BASELINE
- EDGE-A3-TAG-MAPPING-NORMALIZATION-DRYRUN
- LINK-A0-SKELETON-PACKAGE
- CONTRACTS-A2 as ONE-private task

## 4. Reclassification

Completed work reclassified:

- CONTRACTS-A0 -> shared foundation seed draft
- CONTRACTS-A1 -> shared Edge/Link schema draft
- EDGE-SOURCE-AUDIT -> shared edge source audit reference
- EDGE-A0 -> shared edge skeleton draft
- EDGE-A0.1 -> shared edge typecheck baseline draft
- EDGE-A1 -> shared edge connector registry dry-run draft

## 5. New ONE Rule

VANTARIS ONE does not own Edge / Link / Contracts runtime.
VANTARIS ONE owns consumer adapter, business modules, operational UI, analytics, MMS, ESG, CDE, reports and Console integration.
