# UMMS A1 Maintenance Context Model

## 1. Maintenance Management Context

Defines maintenance lifecycle reference context for planning, assignment, and completion states.

## 2. Work Order Reference Context

Defines work order reference context without creating a runtime engine in A1.

## 3. Preventive / Corrective Context

Defines preventive and corrective maintenance reference contexts.

## 4. SLA / Assignment / Escalation Context

Defines SLA, assignment, and escalation linkage references for maintenance orchestration context.

## 5. Spare Parts / Technician / Team Context

Defines spare parts, technician, and team reference contexts.

## 6. UDOC / UESG / UCDE / UCore Linkage

Defines asset/location, energy linkage, evidence linkage, and operation linkage references.

## 7. ONE Adapter Shared Foundation Reference

Defines reference-only consumption boundary through ONE Adapter.

## A1 Boundary

- UMMS does not become source-of-record.
- UMMS does not mutate source system records.
- UMMS does not connect directly to Edge/Link/DB.
- UMMS A1 does not define real DB schema.
- UMMS A1 does not define API contract.
- UMMS A1 only provides future A2 manifest and context foundation.
