# UDOC A1 Operation Context Model

## 1. Site / Building / Floor / Room / Zone context

Defines location hierarchy references for operations visibility and contextual grouping.

## 2. Rack / Cabinet / Asset context

Defines rack/cabinet and related asset placement references for operational alignment.

## 3. Power / Cooling / Environment context

Defines power, cooling, and environment references as operational context objects.

## 4. Topology / Connectivity context

Defines topology and connectivity references for dependency and impact context.

## 5. Alarm / Event reference context

Defines event/alarm reference linkage used for operations and incident context.

## 6. UMMS maintenance linkage

Defines handoff/association references to UMMS maintenance context only.

## 7. UESG energy/sustainability linkage

Defines energy and sustainability linkage references consumed by UESG context flows.

## 8. UCDE evidence linkage

Defines evidence linkage references for UCDE traceability and audit context.

## 9. UCore operational linkage

Defines linkage to UCore operational context for site/asset/event consistency.

## 10. ONE Adapter shared foundation reference

Defines consumption boundary via ONE Adapter for shared foundation reference objects.

## A1 Boundary

- UDOC does not become source-of-record.
- UDOC does not mutate source system records.
- UDOC does not connect directly to Edge/Link/DB.
- UDOC A1 does not define real DB schema.
- UDOC A1 does not define API contract.
- UDOC A1 only provides future A2 manifest and context foundation.
