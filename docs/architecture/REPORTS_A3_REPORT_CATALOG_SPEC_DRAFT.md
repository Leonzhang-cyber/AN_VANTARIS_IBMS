# REPORTS A3 Report Catalog Spec Draft

## 1. A3 Objective

Define Reports report catalog candidate specification at docs level for future implementation gates.

## 2. A2 Gate Reviewed

Reports A2 API/Data Model Gate outputs are reviewed as baseline inputs for A3 catalog candidate drafting.

## 3. Non-scope

- no report engine implementation
- no backend API implementation
- no frontend page implementation
- no DB schema implementation
- no OpenAPI/JSON Schema implementation
- no export runtime implementation
- no scheduled report runtime implementation
- no PDF/Excel generation runtime

## 4. Report Catalog Purpose

Provide a structured, grouped candidate catalog for future reporting capabilities while maintaining docs-only boundaries.

## 5. Catalog Group Model

- Fault & Alarm Reports
- Maintenance Reports
- Energy & Sustainability Reports
- Evidence & Audit Reports
- Data Operations Reports
- Operations Summary Reports
- Module Status Reports
- Governance & Compliance Reports

## 6. First-batch Candidate Summary

A3 defines first-batch candidate reports across all eight groups, including UFMS fault/alarm, UMMS maintenance, UESG sustainability, UCDE evidence, UDOC operations, UCore operations summary, module status, and governance/compliance candidates.

## 7. Metadata Model Summary

A3 defines candidate metadata groups for identity, grouping, source references, filters, aggregation, export, schedule, permission, audit, evidence linkage, and retention.

## 8. Filter/Export/Schedule Candidate Summary

A3 defines filter, aggregation, export format, and schedule eligibility candidates as text-only planning artifacts.

## 9. Forbidden Implementation Work

- creating report engine/runtime
- creating backend/frontend/API/DB implementations
- creating OpenAPI/JSON Schema/contracts artifacts
- creating export/schedule runtime mechanisms

## 10. Next Task

`REPORTS-A4-REPORT-IMPLEMENTATION-GATE-PLAN`

## 11. A3 Declaration

REPORTS-A3 is a docs-level catalog spec draft only. It does not create report engine, backend API, frontend page, DB schema, OpenAPI, JSON Schema, export runtime, scheduled job, or PDF/Excel generation.
