# REPORTS A1 Report Catalog Model

This catalog defines future report candidates as docs-level planning only.
A1 does not create templates, report engine runtime, or export runtime.

## 1) Fault & Alarm Reports

- purpose: summarize and analyze fault/alarm/event references
- source references: UFMS fault/alarm/event references, UCore operational context references
- future filters: site, severity, status, time range, source module
- future aggregation: count/trend/rate and status distribution
- future export options: PDF/Excel/CSV candidates (planning only)
- A1 decision: candidate category only

## 2) Maintenance Reports

- purpose: maintenance workflow and SLA visibility
- source references: UMMS work order and SLA references
- future filters: team, status, priority, asset, date range
- future aggregation: SLA attainment, backlog trend, completion trend
- future export options: PDF/Excel/CSV candidates (planning only)
- A1 decision: candidate category only

## 3) Energy & Sustainability Reports

- purpose: sustainability and energy reporting context visibility
- source references: UESG sustainability/energy references
- future filters: site, metric type, period, source scope
- future aggregation: period totals, trend lines, comparative summaries
- future export options: PDF/Excel/CSV candidates (planning only)
- A1 decision: candidate category only

## 4) Evidence & Audit Reports

- purpose: evidence-linked and audit-oriented context visibility
- source references: UCDE evidence references, governance/audit references
- future filters: evidence type, module, traceability linkage, time range
- future aggregation: traceability summaries and evidence coverage
- future export options: PDF/Excel/CSV candidates (planning only)
- A1 decision: candidate category only

## 5) Data Operations Reports

- purpose: operations and data operations context reporting
- source references: UDOC operations references, UCore context references
- future filters: site, facility area, operation category, period
- future aggregation: capacity/utilization and operation summary trends
- future export options: PDF/Excel/CSV candidates (planning only)
- A1 decision: candidate category only

## 6) Operations Summary Reports

- purpose: unified operational summary across modules
- source references: UCore + module-level standardized references
- future filters: module, site, period, readiness status
- future aggregation: cross-module summary and trend views
- future export options: PDF/Excel/CSV candidates (planning only)
- A1 decision: candidate category only

## 7) Module Status Reports

- purpose: module readiness/dependency visibility
- source references: UConsole module status references
- future filters: module, status, dependency, period
- future aggregation: readiness snapshots and trend comparisons
- future export options: PDF/Excel/CSV candidates (planning only)
- A1 decision: candidate category only

## 8) Governance & Compliance Reports

- purpose: governance and compliance visibility
- source references: governance status and audit references
- future filters: policy, risk category, module, period
- future aggregation: compliance coverage and risk trend summaries
- future export options: PDF/Excel/CSV candidates (planning only)
