# VANTARIS IBMS MySQL to PostgreSQL Migration Plan

## 1. Migration Objective

Move IBMS backend DB architecture from MySQL/PyMySQL toward PostgreSQL while preserving API behavior and system module boundaries.

**Direction change:** MySQL local-smoke is **paused**. PostgreSQL is the canonical target.

---

## 2. Migration Principles

- PostgreSQL as canonical relational DB
- SQLAlchemy ORM-first; raw SQL minimized
- `IBMS_DATABASE_URL` as primary DB config (single source of truth)
- Alembic / Flask-Migrate for migrations (introduce in Phase 5)
- No production data migration until schema is stable
- No `.env` secrets committed
- PostgreSQL local-smoke before production deployment
- UFMS schema/code must not be copied into IBMS
- Smoke DDL is not production migration

---

## 4. MySQL-Specific Items to Replace

| MySQL item | Location | PostgreSQL replacement |
| ---------- | -------- | ---------------------- |
| PyMySQL URL | `default.py`, `database.py` | `postgresql+psycopg://` |
| `mysql+pymysql` hard-code | `database.py` | Use `IBMS_DATABASE_URL` |
| `AUTO_INCREMENT` / BigInteger AI | ORM models | `GENERATED BY DEFAULT AS IDENTITY` |
| `ON DUPLICATE KEY UPDATE` | `menu_api.py` (multiple) | `ON CONFLICT ... DO UPDATE` |
| `GROUP_CONCAT` | `menu_api.py` init-data | `string_agg(..., ',')` |
| `CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | `menu_models.py`, `DID/models.py` | trigger or app `updated_at` |
| `SmallInteger` 0/1 flags | menu/version APIs | `boolean` or keep smallint |
| `dialects.mysql.CHAR/VARCHAR/TINYINT` | `Iot/models.py` | `String`, `SmallInteger` |
| `JSON` column type | multiple models | `JSONB` (PG) via SQLAlchemy |
| Positional `SELECT *` rows | `menu_api.py` | named columns / ORM |

## 5. Stop Conditions

- Production DB credentials required
- Production data import or dump required
- Destructive schema change without rollback plan
- UFMS content detected in IBMS scope
- Broad rewrite without phased approval
- Resuming MySQL smoke without explicit Leon approval
