# UESG A1 Sustainability Context Model

## 1. Energy Operations Context

Defines energy usage reference context and operational baseline references.

## 2. Carbon and Sustainability Context

Defines carbon and sustainability reference context for interpretation and reporting linkage.

## 3. Green Mark and ESG Reporting Context

Defines reporting reference context for Green Mark and ESG deliverables.

## 4. Utility Consumption Context

Defines utility consumption reference context for electricity, cooling, and related utilities.

## 5. Environmental Performance Context

Defines environmental performance reference context used by sustainability analysis.

## 6. Energy Anomaly Context

Defines anomaly reference context for energy and utility behavior.

## 7. UDOC / UMMS / UCDE / UCore Linkage

Defines cross-module linkage references for operations, maintenance, and evidence traceability.

## 8. ONE Adapter Shared Foundation Reference

Defines reference-only consumption boundary through ONE Adapter.

## A1 Boundary

- UESG does not become source-of-record.
- UESG does not mutate source system records.
- UESG does not connect directly to Edge/Link/DB.
- UESG A1 does not define real DB schema.
- UESG A1 does not define API contract.
- UESG A1 only provides future A2 manifest and context foundation.
