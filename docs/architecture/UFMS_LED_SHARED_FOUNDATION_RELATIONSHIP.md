# UFMS-led Shared Foundation Relationship

## 1. UFMS-led Foundation

AN_VANTARIS_Contracts, AN_VANTARIS_EDGE, and AN_VANTARIS_LINK are developed first under UFMS-led shared foundation.

## 2. Why UFMS leads first

- UFMS can be sold independently
- UFMS needs fault/event/alarm ingestion early
- Edge/Link are critical to UFMS value
- ONE can accelerate by consuming shared interfaces later
- shared foundation avoids duplicate development

## 3. VANTARIS ONE Relationship

VANTARIS ONE consumes:

- normalized event/alarm/telemetry objects
- Edge health
- Link delivery state
- contract-defined evidence/audit objects
- UFMS fault intelligence output where required

VANTARIS ONE does not copy:

- UFMS runtime
- Edge protocol runtime
- Link delivery runtime
- UFMS auth/login/seed/migration/schema internals
